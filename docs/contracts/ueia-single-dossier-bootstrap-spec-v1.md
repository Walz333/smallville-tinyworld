## Dossier Purpose

This dossier defines the first constrained bootstrap specification for UEIA archive, packet, brief, and review examples.

Its purpose is to show how one bounded dossier may combine:
- accepted reviewed-run evidence
- accepted control context
- approved static design-asset references

without granting execution rights, runtime mutation authority, or direct world-building authority.

## Dossier Boundaries

This specification uses one constrained dossier only.

The dossier boundary includes:
- one primary operational dossier
- one accepted control context
- one approved static design-asset reference pack

The dossier boundary excludes:
- additional operational runs
- additional asset packs
- live connectors
- scraping
- runtime coupling
- implementation work

## Role Of The Water Mill Context Pack

The Water Mill context pack is included to widen descriptive vocabulary for:
- built features
- water-system elements
- access and maintenance structures
- energy-support features
- off-grid water and power interpretation prompts
- waste or reclassification touchpoints as research prompts

Its contents remain:
- approved static design assets
- snapshot or reference sources only
- non-authoritative for observed real-world status

Its contents must not be treated as:
- observed real-world structures unless separately anchored by accepted source evidence
- runtime geometry
- world-state authority
- implementation instructions

## Single-Dossier Rule

This bootstrap specification is single-dossier only.

It must use:
- one constrained dossier only

It must not use:
- multi-dossier merge
- cross-dossier synthesis unless separately approved

## Provenance And Fact-Status Rule

This specification preserves provenance between:
- accepted run evidence
- raw Water Mill image observation
- later inference
- possible packet use

No derived fact may appear without explicit fact status.

No designed structure may be treated as observed reality unless supported by accepted source evidence.

## Runtime And Execution Boundary

This specification does not authorize:
- runtime launch
- runtime mutation
- automatic approval
- implementation
- direct world or scene mutation

No role in this specification gains execution rights.

## Packet-Is-Not-World-State Rule

Any packet example derived from this dossier is a reviewable interpretive artifact only.

It is not:
- runtime world state
- direct scene mutation authority
- direct world mutation authority
- an implementation plan
