## Why One Specimen Per Family

One specimen per family is promoted because it creates the smallest clean canonical chain with the richest provenance:
- one archive record as the provenance anchor
- one derived packet as the interpretive carrier
- one agent brief as the bounded downstream consumer
- one review decision as the approval-boundary checkpoint

This is sufficient for later tooling and tests without generating new broad example families.
